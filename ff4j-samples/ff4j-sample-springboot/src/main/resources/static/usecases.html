Please find some URL to perform operation on the HR service

Companies

  • List all Companies (No pagination)  Try it
    /hr/v1/companies
  • List all Companies and fetch links to employees (No pagination)  Try it
    /hr/v1/companies?employees=LINK
  • List all Companies and fetch full employees data (No pagination)  Try it
    /hr/v1/companies?employees=CONTENT
  • List companies in a page with 2 elements (on 3)  Try it
    /hr/v1/companiesPageable?page=0&size=2
  • Display a company with employees as link only  Try it
    /hr/v1/companies/1
  • Display a company with employees and departments as link only  Try it
    /hr/v1/companies/1?employees=LINK&departments=LINK
  • Display a company with all data related to employees  Try it
    /hr/v1/companies/1?employees=CONTENT
  • Display only the employees of target Company  Try it
    /hr/v1/companies/1/employees
  • Display only the departments of target Company  Try it
    /hr/v1/companies/1/departments
  • Try to read a company which does not exist  Try it
    /hr/v1/companies/5
  • Create a company (do not know the idCompany)   Swagger
    (POST) /hr/v1/companies
  • Update a company   Swagger
    (PUT) /hr/v1/companies/1
  • Delete a company   Swagger
    (DELETE) /hr/v1/companies/1


Employees

  • List all Employees (No pagination)  Try it
    /hr/v1/employees
  • List a target employee by its id  Try it
    /hr/v1/employees/1
  • Display the company of an employee  Try it
    /hr/v1/employees/1/company
  • List a target that does not exist  Try it
    /hr/v1/employees/8
  • Create a employee (do not know the idEmployee)   Swagger
    (POST) /hr/v1/employees
  • Update an employee   Swagger
    (PUT) /hr/v1/employees/1
  • Delete an employee   Swagger
    (DELETE) /hr/v1/employees/1


Departments

  • List all departments  Try it
    /hr/v1/departments
  • Display 1 department  Try it
    /hr/v1/departments/1
  • Display 1 department with company and employees  Try it
    /hr/v2/departments/2?company=CONTENT&employees=CONTENT
  • Display the company of a department  Try it
    /hr/v1/departments/2/company
  • Display the employees of a department  Try it
    /hr/v1/departments/2/employees
  • Display all departments of a company  Try it
    /hr/v1/companies/1/departments


Communities

  • List all departments of a company (No pagination)  Try it
    /hr/v1/employees
